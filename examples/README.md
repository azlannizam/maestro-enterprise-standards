# Examples

This directory holds **worked examples** that show MES standards applied in
practice — filled-in report structures, sample checklists, and reference
artifacts.

Examples are **illustrative, not normative.** They demonstrate how to satisfy a
standard; they do not add, alter, or override any requirement. Where an example
and a standard appear to disagree, the standard prevails.

## What belongs here

- A completed **Code Review Report** (MES-102) for a sample change.
- A completed **Milestone Completion Report** (MES-103) with an Evidence Index.
- A completed **Deployment Report** (MES-104).
- Any other concrete artifact that helps teams and agents apply a standard.

## What does not belong here

- New requirements or policy (those go in a standard).
- Product-specific implementation detail (that lives in product repositories).

## Naming

Name each example after the standard it illustrates, e.g.
`MES-102_EXAMPLE_CODE_REVIEW_REPORT.md`.

> No examples ship in MES v1.0.0. They will be added as standards are exercised
> on real milestones.
